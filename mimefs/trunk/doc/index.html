mimefs

mimefs is a networked database filesystem, specified as an XMLRPC API. This distribution includes a client library and a server implementation, both written in Python. The server is the de facto definition of the mimefs API.

Introduction

mimefs is a solution to the problem of information storage and retrieval, so it occupies the same space as the filesystem on the one hand, and RDBMSes on the other. The genius of the Unix filesystem is that it is agnostic: interpretations of the meaning of a file's contents are refered to the application layer. The result is flexibility and lower overhead in application development. On the other hand, an RDBMS makes metadata more useful, and provides ACIDity and a sane network abstraction, leading to solid scalability. mimefs offers the best of both worlds: the arbitrary metadata and robust transactions of an RDBMS, and the flexibility of a filesystem.

To whet your whistle, here is a "hello world" example using mimefslib, the Python client:

>>> from mimefslib import Volume
>>> vid = "228434f2-885f-11da-8e59-00b0d091c186"
>>> vol = Volume("http://localhost:5370/%s" % vid)
>>> msg="""\
... Content-Type: x-my-app/foo
... x-bar: baz
...
... Greetings, program!
... """
>>> mid = vol.open(create=True)
>>> vol.write(mid, msg)
>>> mids = vol.list('x-bar = baz')
>>> mids
['f6e700b8-0aa1-4e9d-8364-a91fd0a38fdf']
>>> msg2 = vol.read(mids[0])
>>> msg == msg2
True

Notice the use of UUIDs to identify both volumes and messages. UUIDs are the new inode. Notice also the familiar open/read/write API. mimefs tries to mimic the traditional stdio API, while also exposing a more database-like API (store, find).
