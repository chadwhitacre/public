Dear Sir or Madam:

Greetings! Thank you for your interest in httpy, a sane Python webserver. httpy exists to bridge your Python application with other HTTP applications, e.g., proxy servers and web browsers.

The default application included with httpy will serve static files off the filesystem, so httpy is useful without configuration out of the box. Assuming it is already installed, here is httpy's "hello world:"

% echo hello world > index.html && httpy

Of course, httpy does expose a simple set of configuration options, which can be flexibly set from the command line, a configuration file, or the environment.

And if you want to do more than serve static files off the filesystem, httpy exposes a sane and practical API to your custom application.

Thank you again for your interest. I hope you find much success with httpy!
